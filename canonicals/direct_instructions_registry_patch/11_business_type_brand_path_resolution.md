# Business Type and Brand Path Resolution Direct Patch

## Business-Type-First Enforcement
For any brand-targeted or business-type-targeted task, resolve the Business Activity Type before reading or writing brand or business knowledge.

The runtime must use:

1. `Business Activity Type Registry`
2. `Business Type Knowledge Profiles`
3. path resolver JSON assets
4. Registry Surfaces Catalog
5. graph and validation surfaces

## No Legacy Path Completion
Do not complete a Business Type or Brand addition using:

- GitHub as final knowledge storage
- My Drive as final knowledge storage
- orphaned Google Drive folders
- markdown placeholders
- `Knowlege/Business-Type`

These may only appear as superseded evidence or migration notes.

## New Business Type Completion
A new Business Type addition must produce:

- canonical Shared Drive folder
- standard folder structure
- editable Google Docs
- Business Activity Type Registry row
- Business Type Knowledge Profiles row
- Surface Catalog row
- Knowledge Graph node
- Relationship Graph relationships
- Validation row
- JSON storage map

## New Brand Under Business Type Completion
A new brand addition must produce:

- resolved Business Type reference
- brand folder under the resolved Business Type
- editable Brand Core docs
- Brand Registry row
- Brand Core Registry rows
- graph bindings
- validation record
- JSON brand path map
